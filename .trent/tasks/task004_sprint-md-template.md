# Task 004: Create SPRINT.md template

## Objective
Create `template_v2/.trent/SPRINT.md` — the active work queue file that the cleanup agent generates nightly, containing a prioritized slice of ≤15 tasks ready for autonomous sprint agents to execute without human supervision.

## Context
The autonomous multi-agent workflow requires a distilled, safe-to-execute queue. The full TASKS.md is too large and contains tasks that need human approval. SPRINT.md is the curated slice — only `ai_safe: true` tasks that are unblocked, have task files, and are within blast_radius tolerance.

## Acceptance Criteria
- [ ] File exists at `template_v2/.trent/SPRINT.md`
- [ ] Contains ≤15 task slots (configurable via PROJECT_CONTEXT.md)
- [ ] Each entry includes: task_id, title, subsystem, priority, estimated_effort, claim_ttl_minutes, blast_radius
- [ ] Header section shows sprint metadata: generated_at, valid_until, generating_agent
- [ ] Footer section shows exclusion summary (tasks skipped and why: not ai_safe, resource-gated, requires_solo_agent, etc.)
- [ ] File is clearly marked as auto-generated (DO NOT EDIT MANUALLY)
- [ ] Includes a "SPRINT RULES" section that agents read before claiming tasks

## Template Content

```markdown
# SPRINT.md — Active Work Queue
<!-- AUTO-GENERATED by @trent-cleanup — DO NOT EDIT MANUALLY -->
<!-- Edit .trent/TASKS.md to change what appears here -->

**Generated**: {YYYY-MM-DD HH:MM UTC}
**Valid Until**: {YYYY-MM-DD HH:MM UTC} (expires in 2h)
**Generated By**: {agent_id}
**Project**: {project_name from PROJECT_CONTEXT.md}

---

## Sprint Rules (Read Before Claiming)

1. Claim a task by updating its YAML: `claimed_by: {your_agent_id}`, `claimed_at: {ISO timestamp}`
2. Commit the claim immediately: `git commit -m "claim(task-{id}): {agent_id} starting work"`
3. If you cannot complete a task within `claim_ttl_minutes`, release the claim: set `claimed_by: null`
4. You MUST NOT modify tasks not in this sprint list without human approval
5. Tasks marked `requires_solo_agent: true` must only be run if you are the ONLY active agent
6. Complete = file updated to `status: in_progress` → work done → file updated to `status: awaiting_verification`
7. Never mark a task `status: completed` — that requires a DIFFERENT agent to verify

---
```

## Implementation Notes
- Template should ship as a blank example with placeholder values in `{curly_braces}`
